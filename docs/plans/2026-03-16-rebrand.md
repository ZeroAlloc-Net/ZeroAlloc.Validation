# ZeroAlloc.Validation Rebrand Implementation Plan

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** Rename every occurrence of `ZValidation` → `ZeroAlloc.Validation` and `ZValidationInternal` → `ZeroAlloc.Validation.Internal` across all source, test, project, and documentation files.

**Architecture:** Five sequential tasks: (1) directory/file renames via `git mv`, (2) project and solution file content, (3) source namespaces and generator FQN strings (must be atomic — these must land in one commit or the build will be broken), (4) test files, (5) documentation. Tasks 1–4 must happen in order; task 5 is independent.

**Tech Stack:** C# / .NET, git, dotnet CLI. All string replacements are mechanical find-replace; no logic changes.

---

> ⚠️ **Critical ordering rule for find-replace**: Always replace `ZValidationInternal` → `ZeroAlloc.Validation.Internal` **before** replacing `ZValidation` → `ZeroAlloc.Validation`. If done in reverse order, `ZValidationInternal` becomes `ZeroAlloc.ValidationInternal` (wrong) and the second replace never fires correctly.
